家庭 AI 管家系統

專案規劃與軟體架構文件

版本 2.0 ｜ 2026 年 3 月

1. 專案概述

本專案目標是打造一個以 Line Bot 為主要介面、Google Sheets 為資料庫、Claude API 為大腦的家庭 AI 管家系統。使用者只需要會用 Line，即可完成所有操作，無需安裝任何額外 App 或學習新工具。

核心設計哲學：

  • 零學習成本：介面就是每天在用的 Line，會用 Line 就會用這個系統
  • 自然語言輸入：說人話，不需要固定格式
  • 主動推播：系統主動告知，使用者不需要主動查詢
  • 資料透明：所有資料存在 Google Sheets，隨時可直接查看與編輯，沒有黑盒子
  • 零廠商鎖定：每個元件都可替換，沒有單點依賴
  • 接近零成本：每月費用約 NT$ 10~20 元

3. 系統架構

3.1 被動接收訊息流程

你傳訊息給 Line Bot
↓
Line 伺服器透過 Webhook 轉發到 Render Server
↓
Render Server 讀取食品庫存、待辦事項、對話歷史
↓
呼叫 Claude API 解析自然語言，回傳 JSON 指令 + 管家語氣回覆
↓
執行對應動作，寫入 Google Sheets
↓
Line Bot 以管家語氣回覆確認訊息

3.2 主動推播流程（每日排程）

每天早上 10 點，GAS 定時觸發
↓
呼叫 Render Server 的 /notify 端點
↓
讀取 Google Sheets，篩選過期食品與本週待辦
↓
公開待辦推播給全家人，私人待辦只推播給負責人
↓
透過 Line Messaging API 主動推播

3.3 即時提醒流程（每 15 分鐘）

每 15 分鐘，GAS 定時觸發
↓
呼叫 Render Server 的 /notify_realtime 端點
↓
檢查未來 20 分鐘內有時間的待辦事項
↓
推播提醒給對應的家庭成員

5. 自然語言處理設計

5.1 Claude API 支援的 action

食品庫存

action說明欄位
add_food新增食品name, quantity, unit, expiry
delete_food食品全部消耗，移至封存name
modify_food修改食品資料name，選填：new_name, quantity, expiry
query_food查詢食品庫存無

待辦事項

action說明欄位
add_todo新增待辦item, date，選填：time, person, type
modify_todo修改待辦item，選填：new_item, date, time, person, type
delete_todo標記完成，移至封存item
query_todo查詢待辦（私人只顯示自己的）無
unclear語意不清時反問message

5.2 回傳 JSON 格式

{"actions": [...], "reply": "用管家語氣寫給使用者看的回覆，自然、簡潔、有禮"} 範例： {"actions": [{"action": "add_food", "name": "牛奶", "quantity": 1, "unit": "瓶", "expiry": "2026-03-25"}], "reply": "好的，牛奶已為您登記，過期日 3 月 25 日。"} {"actions": [{"action": "modify_food", "name": "橘子", "quantity": 2}], "reply": "好的，橘子已更新為 2 個。"} {"actions": [{"action": "modify_todo", "item": "看牙醫", "date": "2026-04-25"}], "reply": "好的，看牙醫已改到 4 月 25 日。"}

5.3 Claude 每次可見的 Context

  • 家庭成員名稱與稱謂
  • 目前食品庫存清單（有效的品項）
  • 目前待辦事項清單（待辦的事項）
  • 今天日期與現在時間
  • 該用戶最近 6 則對話歷史

6. 自然語言輸入範例

6.1 食品管理

使用者說法系統理解
「牛奶 3/25」新增牛奶，過期日 2026/3/25
「剛買的雞蛋，大概一週後過期」新增雞蛋，推算過期日
「橘子吃掉1個」橘子數量減少 1 個
「橘子改成 4 個」橘子數量更新為 4 個
「牛奶喝完了」語意解析，刪除牛奶
「牛奶名稱改成鮮奶」修改品名

6.2 待辦事項

使用者說法系統理解
「下週三下午看醫生」新增待辦，計算具體日期時間
「提醒大家下週五家庭聚餐」新增公開待辦
「看牙醫改到下週五」修改看牙醫的日期
「把看牙醫改成洗牙」修改待辦名稱
「看牙醫完成了」標記完成，移至封存

6.3 每日推播格式範例

  • 🔴 今天到期：牛奶
  • 🟡 3天內到期：優格（3/15）、起司（3/16）

8. 問題排查指南

問題現象排查方向
Line 傳訊息沒反應查看 Render Log，確認 Webhook 是否有收到請求；第一則訊息冷啟動可能失敗，重傳一次
Bot 回覆「系統錯誤」查看 Render Log 的完整 traceback 找出錯誤位置
資料沒有寫進 Sheets查看 Google Sheets API 錯誤訊息，確認 Service Account 有編輯權限
每日通知沒收到確認 GAS 排程是否正常觸發，查看 GAS 執行記錄
Bot 理解錯誤修改 SYSTEM_PROMPT，更明確描述要的行為，push 後重新部署
UptimeRobot 顯示 DownServer 已支援 HEAD 請求，顯示 Down 通常是誤判，實際服務正常

9. 架構擴展性

本系統底層為通用家庭管家框架，所有功能共享相同技術基礎：

Line Bot（介面）+ Claude API（大腦）+ Google Sheets（記憶體）

新增任何功能只需三步驟：

  • 新增對應的 Google Sheets 分頁
  • 撰寫對應的 handle 函數與排程推播邏輯
  • 更新 System Prompt 加入新功能描述

未來可擴充的功能包括：採購清單、繳費提醒、洗衣提醒、冰箱 Dashboard 等。無需改動核心架構，完全向後相容。

本文件根據實際開發過程持續更新，版本 2.0 反映 2026 年 3 月的完整實作狀態，包含食品封存、待辦封存、對話封存、modify_food、modify_todo 等功能。
